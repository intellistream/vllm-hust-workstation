# 2. 总体技术架构

当前确认的架构形态：

- `vllm-hust`：统一推理底座
- `workstation`：本地控制面、指标聚合、任务入口
- `EvoScientist`：首个多智能体真实业务测试对象
- OpenAI-compatible 接口：统一上层接入协议
- Plugin / Platform：统一后端与国产算力适配边界

核心能力已经具备：

- 多模态输入处理
- 流式输出与 reasoning 输出
- 工具调用与结构化输出
- 指标采集与 benchmark 链路
- Kubernetes / Helm / 本地工作站两类部署入口

当前新增落地：

- EvoScientist 已默认对接本地 `vllm-hust`
- `workstation -> EvoScientist -> vllm-hust` 已是默认桥接链路

# 5. 风险与边界

会上要主动讲清楚四点：

1. 多智能体方面

- 现在是“已有真实业务测试对象”
- 不是“通用多智能体仿真平台已完成”

2. 流式强化学习方面

- 现在完成的是 serving 依托和数据面方向
- 不是 RL 训练闭环已经落地

3. 云原生调度方面

- 当前强项是部署与运行形态
- 不是跨集群调度算法验证已经完成

4. 国产算力方面

- 瓶颈方向已经识别
- 但还需要补跨硬件实验数据支撑
